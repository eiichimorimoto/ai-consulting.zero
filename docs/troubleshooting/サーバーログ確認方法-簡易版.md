# サーバーログの確認方法（簡易版）

## 📋 サーバーログとは？

Next.jsの開発サーバー（`npm run dev`）を起動したターミナルに表示されるメッセージのことです。
エラーが発生すると、このログに詳細な情報が表示されます。

## 🔍 ログを確認する手順

### 方法1: 開発サーバーを起動したターミナルを確認（最も簡単）

1. **ターミナルアプリケーションを開く**
   - Macの場合: `Command + Space` でSpotlightを開き、「ターミナル」と入力
   - または、`アプリケーション` → `ユーティリティ` → `ターミナル`

2. **開発サーバーが起動しているウィンドウを探す**
   - `npm run dev` を実行したターミナルウィンドウを探してください
   - そのウィンドウに以下のようなメッセージが表示されているはずです：
     ```
     ▲ Next.js 16.1.0
     - Local:        http://localhost:3000
     - Ready in 2.3s
     ```

3. **エラーが発生したら、そのターミナルウィンドウを見る**
   - ブラウザでエラーが発生したら、すぐにターミナルウィンドウに注目
   - 以下のようなログが表示されます：
     ```
     🔑 PageSpeed APIキー確認: { hasKey: true, keyLength: 39, ... }
     📡 PageSpeed API呼び出し (mobile): { url: '...', ... }
     ❌ PageSpeed API Error (mobile): { status: 403, ... }
     ```

### 方法2: 新しいターミナルでログを確認

開発サーバーがバックグラウンドで実行されている場合：

1. **新しいターミナルウィンドウを開く**

2. **プロジェクトディレクトリに移動**
   ```bash
   cd /Users/eiichi/Documents/ai-consulting-zero
   ```

3. **ログファイルを確認（もしあれば）**
   ```bash
   # .next/trace ファイルがあれば
   tail -f .next/trace
   ```

### 方法3: ブラウザの開発者ツールで確認

1. **ブラウザで開発者ツールを開く**
   - Mac: `Command + Option + I`
   - Windows/Linux: `F12` または `Ctrl + Shift + I`

2. **「Console」タブを開く**
   - エラーメッセージが表示されます

3. **「Network」タブを開く**
   - APIリクエストの詳細を確認できます
   - エラーが発生したリクエストをクリックして詳細を確認

## 📝 確認すべきログの例

### PageSpeed API関連のログ

```
🔑 PageSpeed APIキー確認: { hasKey: true, keyLength: 39, keyPrefix: 'AIzaSyDQ6s...' }
📋 URL正規化: { original: 'http://www.ootake.co.jp', normalized: 'http://www.ootake.co.jp/' }
🔍 PageSpeed分析開始: { url: 'http://www.ootake.co.jp/', ... }
📡 PageSpeed API呼び出し (mobile): { url: '...', encodedUrl: '...', ... }
❌ PageSpeed API Error (mobile): { status: 403, statusText: 'Forbidden', errorText: '...' }
```

### エラーログの見方

- `🔑`: APIキーの確認
- `📋`: URLの正規化
- `🔍`: 分析開始
- `📡`: API呼び出し
- `❌`: エラー発生

## ⚠️ ログが見つからない場合

### 開発サーバーが起動していない場合

新しいターミナルで：
```bash
cd /Users/eiichi/Documents/ai-consulting-zero
npm run dev
```

### ログが多すぎる場合

特定のエラーのみを確認：
```bash
# ターミナルで Command + F を押して検索
# 「PageSpeed」や「Error」で検索
```

## 💡 ヒント

- **ログはリアルタイムで更新されます** - ブラウザで操作すると、すぐにターミナルにログが表示されます
- **エラーが発生したら、すぐにターミナルを見る** - エラーメッセージは数秒で消える場合があります
- **ログをコピーする** - エラーログを選択してコピー（`Command + C`）して、問題解決に役立ててください
