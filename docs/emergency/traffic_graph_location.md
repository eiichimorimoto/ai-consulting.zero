# 🔍 「レスポンスコード別のトラフィック」グラフの探し方

## 📊 グラフの位置

現在見ているページ（APIサービスの詳細）で、**上にスクロール**してください。

グラフは以下の順番で並んでいます：

```
1. レスポンスコード別のトラフィック  ← これが最重要！
   ↓
2. APIメソッド別のエラー数        ← 現在ここを見ている
   ↓
3. 全体的なレイテンシ
   ↓
4. APIメソッド別のレイテンシ（中央値）
```

---

## 🎯 「レスポンスコード別のトラフィック」グラフで確認すべきこと

### グラフの見方

```
縦軸: リクエスト数
横軸: 時刻

色分け:
- 緑色（200）: 成功したリクエスト
- 赤色（400番台）: クライアントエラー（429含む）
- オレンジ（500番台）: サーバーエラー
```

### 確認すべきポイント

1. **今日の合計リクエスト数**
   - グラフの棒の高さ
   - 数値が表示される

2. **400エラー（赤色）の数**
   - 429エラーはここに含まれる
   - どのくらいの割合か

3. **急増したタイミング**
   - 特定の時間帯に集中しているか
   - 一定のペースか
