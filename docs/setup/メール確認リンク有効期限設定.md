# メール確認リンクの有効期限設定

## 📧 概要

メール確認リンクの有効期限を1時間に延長する設定方法です。

## 🔧 Supabaseダッシュボードでの設定（推奨）

### ステップ1: Supabaseダッシュボードを開く

1. [Supabaseダッシュボード](https://supabase.com/dashboard)にアクセス
2. プロジェクト「ai^consulting-zero」を選択

### ステップ2: 認証設定を開く

1. 左メニューから「**Authentication**」→「**Settings**」を開く
2. 「**Email Auth**」セクションをスクロール

### ステップ3: 有効期限を設定

1. 「**Email confirmation link expiry**」または「**Confirmation link expiry**」を探す
2. 有効期限を**1時間（3600秒）**に設定
   - 単位が秒の場合: `3600`
   - 単位が分の場合: `60`
   - 単位が時間の場合: `1`

### ステップ4: 保存

1. 下にスクロールして「**Save**」をクリック
2. 設定が反映されるまで数秒待つ

## ⚠️ 注意事項

- 有効期限を長くすると、セキュリティリスクが若干増加します
- 1時間は一般的な設定値で、多くのサービスで採用されています
- 設定変更後、新しいサインアップで有効期限が反映されます

## 🔍 確認方法

1. 新しいユーザーでサインアップ
2. メール内の確認リンクを確認
3. 1時間以内にリンクをクリックして、正常に動作することを確認

## 📝 参考

- Supabase公式ドキュメント: [Email Auth Settings](https://supabase.com/docs/guides/auth/auth-email)
- デフォルトの有効期限は通常、数分〜数十分に設定されています
